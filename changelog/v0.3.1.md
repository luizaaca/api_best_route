# Release v0.3.1

## Summary

This version tightens the console lab configuration semantics, makes operator overrides deterministic, and turns `output.verbose` into a real end-to-end runtime signal for the console lab workflow.

## Included Changes

### Console lab configuration precedence

- updated explicit-mode config expansion so operator blocks are replaced as whole objects instead of being deep-merged;
- applied whole-object precedence to:
  - `population_generator`;
  - `selection`;
  - `crossover`;
  - `mutation`;
- prevented stale operator `params` from leaking through defaults when an experiment overrides only the operator `name`.

### Operator param handling

- updated lab operator factories so unsupported params on known operators no longer raise exceptions;
- changed the console lab behavior to ignore unsupported params and keep the session running;
- preserved hard failures for unknown operator identifiers.

### Verbose runtime flow

- added verbose-aware runtime logging across the console lab orchestration flow;
- extended `output.verbose` to cover:
  - config loading and run expansion;
  - shared graph and adjacency setup;
  - optimizer composition;
  - ignored-param notices;
  - genetic-algorithm progress and final summaries.

### Documentation

- updated `lab/README.md` to document:
  - whole-object operator override precedence;
  - supported params per operator;
  - ignored unsupported params for known operators;
  - the expanded meaning of `output.verbose`;
- updated `README.md` with a concise summary of the new lab-mode behavior.

## Notes

- the new ignored-param behavior applies only to known operators in console lab mode;
- unknown operator names remain configuration errors;
- API behavior is intentionally unchanged in this release.
