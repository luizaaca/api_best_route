# Release v0.4.2

## Summary

This release updates documentation and usability notes in preparation for general stability improvements, including explicit console CLI argument docs and beta warnings for lab grid/random modes.

## Included Changes

### Console docs update

- enhanced `README.md` console example section with explicit usage of:
  - `--max-generation`
  - `--max-processing-time`
  - existing `--verbose`
- added usage examples for command-line max generation/time parameters.

### Lab mode beta warning

- added beta stability notice to `lab/README.md` in both:
  - `Grid Mode`
  - `Random Mode`
- clarifies that these modes may behave unpredictably on complex/edge-case scenarios.

### Adaptive GA reseed reliability

- promoted `injection_size` from `GenerationOperators.metadata` to a first-class operator field.
- added state-level `injection_size` support in `build_route_generation_operators`.
- enforced `injection_size <= population_size // 2` in `GeneticAlgorithm._resolve_injection_size`, ignoring and logging oversize values.
- changed engine update loop to preserve elite + build offspring for `population_size - injection_size` before applying injection slots.
- added tests in:
  - `tests/test_genetic_algorithm_state_controller.py`
  - `tests/test_genetic_algorithm_builders.py`
- updated example config `console/example.config.json` to `injection_size` top-level key.

## Main Files Impacted

- `README.md`
- `lab/README.md`

## Validation

- documentation-only change; no code runtime changes.

## Notes

- no behavior changes in code paths; only user-facing documentation updates.
