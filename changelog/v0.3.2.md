# Release v0.3.2

## Summary

This version consolidates the route-domain GA composition seam by introducing shared infrastructure builders for adaptive state-controller assembly and a shared `TSPOptimizerFactory` used by the API and console entry points.

## Included Changes

### Shared TSP optimizer composition

- added `src/infrastructure/tsp_optimizer_factory.py`;
- centralized `TSPGeneticAlgorithm` construction behind `TSPOptimizerFactory.create()`;
- updated:
  - `api/dependencies.py`;
  - `console/main.py`;
  - `console/lab/orchestration/lab_optimizer_builder.py`;
  to delegate final optimizer assembly to the shared factory.

### Shared GA builder package

- added `src/infrastructure/genetic_algorithm/builders/` as the shared home for reusable composition helpers;
- extracted shared population-distance resolution into `distance_strategy_builder.py`;
- extracted shared operator and specification resolution into the builder package;
- extracted route adaptive state-controller assembly into `adaptive_state_controller_builder.py`.

### API adaptive composition cleanup

- removed the inline adaptive state-controller assembly logic from `api/dependencies.py`;
- kept `api/config.json` as the required adaptive configuration source;
- preserved the API request schema and config-only behavior.

### Tests

- added focused builder regression coverage in `tests/test_genetic_algorithm_builders.py`;
- validated shared composition changes against:
  - `tests/test_genetic_algorithm_builders.py`;
  - `tests/test_console_lab.py`;
  - `tests/test_route_service.py`;
  - `tests/test_tsp_genetic_algorithm.py`.

- this release does not redesign the lab configuration schema yet;
- the shared composition seam is an intermediate step toward the later problem-family / abstract-factory architecture.
