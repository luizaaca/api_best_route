# Release v0.4.1

## Summary

This release refines the adaptive-state transition model, makes transition reporting explicit in verbose mode, and adds CLI support for the console generation limits.

## Included Changes

### Adaptive transition semantics

- removed the run-relative `progress_at_least` specification from the public transition vocabulary;
- introduced a state-local improvement model based on explicit state-entry baselines;
- replaced the previous-generation `improvement_below` rule with `window_improvement_below`;
- renamed the stale-count rule to `no_improvement_for_generations` and made it explicitly state-local;
- updated the adaptive state controller and engine so transition resolution carries explicit source and target state data.

### Verbose transition reporting

- made transition events visible in verbose mode through the application and console hooks;
- added explicit transition messages that include the generation number, transition label, source state, and target state;
- kept the generic GA engine logs focused on orchestration and removed the per-generation verbose shortcut from the engine layer.

### Console CLI updates

- added `--max-generation` and `--max-processing-time` to the console entry point;
- wired those arguments into the interactive console example execution path;
- left lab runs config-driven, with verbose output still controlled by the lab JSON configuration.

### Test and config alignment

- updated the adaptive builder and controller tests for the new explicit transition semantics;
- updated the console and route-service tests for the new generation-record shape and transition messages;
- migrated the example and explicit lab configs to the new state-local specification names.

## Notes

- this release intentionally breaks the old transition-spec vocabulary in favor of a clearer state-local model;
- verbose transition messages are now emitted by the outer orchestration layers rather than by the generic GA engine itself;
- the console CLI arguments only affect the interactive example path, while lab behavior remains config-driven.
