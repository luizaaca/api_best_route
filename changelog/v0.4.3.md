# Release v0.4.3

## Summary

This release improves runtime observability for the API and route optimization service by wiring shared logger injection, replacing console prints with structured log calls, and documenting debug-level usage.

## Included Changes

### API request payload logging

- added a request middleware in `api/main.py` to inspect incoming payload bodies;
- logs request payload content through the module logger at `debug` level;
- keeps payload logging opt-in through the uvicorn log level.

### Shared logger injection

- injected a shared logger from `api/dependencies.py` into `RouteOptimizationService`;
- keeps the application composition root responsible for logger wiring;
- preserves the existing service contract while enabling terminal-visible runtime logs.

### Route service log cleanup

- replaced `print()` calls in `src/application/route_optimization_service.py` with `self._log(...)`;
- keeps initialization, optimization, and coordinate-conversion messages on the configured logger;
- normalized `logging.Logger` inputs so the service accepts either callable loggers or logger instances.

### README debug example

- added a `uvicorn` debug-level example to `README.md`;
- documents how to enable request payload logs and internal service trace messages from the terminal.

## Main Files Impacted

- `api/main.py`
- `api/dependencies.py`
- `src/application/route_optimization_service.py`
- `README.md`

## Validation

- verified changes with the repository test suite using the project virtual environment;
- confirmed the API can be launched with uvicorn debug logging enabled.

## Notes

- request payload logging is intentionally emitted at `debug` level to avoid noisy default output;
- the logger change is scoped to the API composition path and route optimization service for this release.
