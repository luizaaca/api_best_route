# Release v0.2.2

## Summary

This version completes the documentation of the modular GA operator library, adding the full set of selection, crossover, and mutation strategy implementations to the architecture specification and README, and introduces an educational concepts suite covering each operator family with theory, bare-metal examples, and domain class usage.

## Included Changes

### GA operator library — complete implementation set

All four concrete implementations per operator category are now present in `src/infrastructure/genetic_algorithm/`:

**Selection (`selection/`)**
- `RoulleteSelectionStrategy` — fitness-proportionate selection via inverse-fitness weights, using numpy normalization.
- `RankSelectionStrategy` — sorts the population by fitness and assigns integer weights inversely proportional to rank, eliminating dominance by a single high-fitness individual.
- `StochasticUniversalSamplingSelectionStrategy` — lower-variance roulette variant; places two evenly-spaced pointers on the cumulative weight distribution and resolves both in a single pass.
- `TournamentSelectionStrategy` — samples `tournament_size` competitors without replacement and elects the best; `tournament_size` is injectable and validated (minimum 2).

**Crossover (`crossover/`)**
- `OrderCrossoverStrategy` — copies a contiguous slice from Parent 1 and fills the remainder in the order they appear in Parent 2.
- `CycleCrossoverStrategy` — traces position-to-position cycles between both parents and alternates which parent each cycle is sourced from.
- `PartiallyMappedCrossoverStrategy` — copies a middle segment from Parent 1 and resolves conflicts in the remainder via iterative inverse-mapping.
- `EdgeRecombinationCrossoverStrategy` — builds a unified adjacency edge map from both parents and constructs the child by greedily extending the neighbour with the fewest remaining connections.

**Mutation (`mutation/`)**
- `InsertionMutationStrategy` — removes one destination from its route and re-inserts it at a random position, optionally crossing vehicle routes.
- `InversionMutationStrategy` — reverses the sub-sequence between two random indices within a single route.
- `TwoOptMutationStrategy` — same reversal operation as inversion, motivated by 2-opt local search: reversing a path segment eliminates crossing edges.
- `SwapAndRedistributeMutationStrategy` — applies one or both of two sub-operations per call: cross-vehicle node redistribution (`_mutate_distribution`) and within-route position swap (`_mutate_vehicle_order`).

### Documentation updates

- updated the GA Architecture class diagram in `architecture.md` to show all four concrete implementations under `ISelectionStrategy`, `ICrossoverStrategy`, and `IMutationStrategy`.
- added an **Available GA Operators** table to `README.md` listing all implementations with a brief description of each.

### Educational concepts notebooks

Added three interactive notebooks under `concepts/` covering each operator family:

- `concepts/crossover_operator_examples.ipynb` — OX, CX, PMX, and ERX crossover operators; each section includes a theoretical explanation, a bare-metal Python implementation on string lists, and a domain class call.
- `concepts/mutation_operator_examples.ipynb` — Insertion, Inversion, Two-Opt, and Swap-and-Redistribute mutation operators; same Theory → Bare-metal → Domain class structure.
- `concepts/selection_operator_examples.ipynb` — Roulette, Rank, SUS, and Tournament selection operators; demonstrates the interface using a mini-population of five `Individual` objects with synthetic `FleetRouteInfo` fitness values.
