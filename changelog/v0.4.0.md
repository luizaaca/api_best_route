# Release v0.4.0

## Summary

This version reorganizes the project documentation into focused references and records the latest route-execution migration to execution bundles plus the generic runner.

## Included Changes

### Documentation refactor

- shortened `README.md` into a concise project entry point;
- kept setup, run instructions, and configuration-file guidance in the root README;
- split the deeper technical content into dedicated references:
  - `architecture.md` for high-level architecture and design patterns;
  - `generic_ga.md` for the generic GA runtime;
  - `routes_optimization.md` for the route-domain models and pipeline;
- preserved `lab/README.md` as the detailed lab contract and linked to it from the root README.

### Bundle and runner composition

- moved route execution to `RouteGAExecutionBundle` plus `GeneticAlgorithmExecutionRunner`;
- kept `RouteOptimizationService` as the application-layer orchestrator while removing route-optimizer construction from the runtime path;
- updated the main composition points to the bundle-first flow:
  - `console/main.py`;
  - `api/dependencies.py`;
  - `console/lab/orchestration/lab_optimizer_builder.py`;
  - `src/application/route_optimization_service.py`;
- narrowed `TSPOptimizerFactory` to execution-bundle composition responsibilities.

### Tests and validation alignment

- updated the affected tests to the new bundle/runner execution contract;
- preserved generation-record capture and runtime logging behavior through the generic runner callbacks;
- aligned the release documentation with the current architecture.

## Notes

- the documentation split is intentionally root-level so the main README can act as the project front door;
- the route-execution bundle and generic runner are now treated as the stable execution boundary for route GA runs;
