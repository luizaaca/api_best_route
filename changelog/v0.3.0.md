# Release v0.3.0

## Summary

This version documents the console lab mode as a first-class benchmark workflow, adds ready-to-run configuration examples for all three execution modes, and clarifies the current configuration contract, including the shared `output` block and the policy-only `random_search` design.

## Included Changes

### Console lab mode documentation

- added `lab/README.md` as the dedicated guide for console lab mode;
- documented the real console entry command implemented in `console/main.py`:
  - `python -m console.main lab --config <path>`;
- described the top-level configuration contract shared across all lab sessions;
- documented the `output` block with:
  - `plot`;
  - `verbose`;
  - `show_best_run_details`;
- documented the three execution styles:
  - explicit mode;
  - grid mode;
  - random mode.

### Random-search contract clarification

- documented that `mode=random` requires `random_search` and forbids:
  - top-level `search_space`;
  - `defaults`;
- documented the required `random_search` fields:
  - `n`;
  - `allowed_generators`;
  - `allowed_selection`;
  - `allowed_crossover`;
  - `allowed_mutation`;
  - `ranges`;
- documented the required scalar range keys:
  - `population_size`;
  - `mutation_probability`;
  - `max_generation`;
  - `max_processing_time`;
  - `vehicle_count`;
- documented that fixed random values are represented explicitly through:
  - singleton operator allow-lists;
  - degenerate ranges where `min == max`.

### Example configurations

Added three root-level lab examples under `lab/`:

- `lab/explicit.config.json` — explicit benchmark session with hand-picked runs;
- `lab/grid.config.json` — Cartesian grid over several GA dimensions;
- `lab/random.config.json` — policy-driven random search using the current typed contract.

These examples are intended as reusable templates for real console experimentation.

### Root documentation

- updated `README.md` to link directly to the new lab guide;
- added `lab/` to the documented project layout;
- added a concise lab-mode section focused on how to run it and where to find the full guide.

### Architecture review

- reviewed `architecture.md` with a src-focused lens;
- no src-related architectural statements required changes in this release, so the file was intentionally left unchanged.

Validated in this documentation slice:

- `architecture.md` was reviewed and intentionally left unchanged because this release does not alter `src/` architecture.

## Notes

- plotting in multi-run lab sessions can distort elapsed-time comparisons;
- all runs in one lab session must share `origin`, `destinations`, `weight_type`, and `cost_type`;
- only JSON lab configuration files are supported.
